# Security policy

## Supported versions

We support the current main branch for security updates. Older branches are not maintained.

## Reporting a vulnerability

If you believe you have found a security vulnerability in this project, please report it privately rather than opening a public issue.

**How to report:**

- Email the maintainers (or open a private security advisory on the repository if the project is hosted on GitHub/GitLab) with a description of the issue, steps to reproduce, and any impact you are aware of.
- Do not disclose the vulnerability publicly until it has been addressed and a release has been made (if applicable).

**What to expect:**

- We will acknowledge receipt and work to confirm and understand the issue.
- We will provide an update on next steps and, if applicable, a timeline for a fix and disclosure.
- We ask that you do not exploit the vulnerability or share it with others before it is resolved.

**Out of scope:**

- Issues that require access to your own SkyFi API key or credentials (keep those secret; never commit them).
- General hardening or best-practice suggestions that are not exploitable vulnerabilities (those can be raised as normal issues or discussions).

Thank you for helping keep this project and its users safe.
